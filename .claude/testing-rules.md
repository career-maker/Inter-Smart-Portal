# Testing Rules

Project: Intersmart Employee Portal

---

## Testing Philosophy

Every feature must be tested.

No feature is complete without tests.

---

## Required Tests

- Unit Tests
- Feature Tests
- Integration Tests
- Permission Tests
- UI Tests

---

## Critical Workflows

Test:

- Login
- Leave Application
- Leave Approval
- WFH Approval
- Document Requests
- Notifications

---

## Validation Testing

Test:

- Empty Fields
- Invalid Inputs
- Duplicate Records
- Unauthorized Access

---

## Permission Testing

Verify:

Employee cannot access Admin pages.

Team Lead cannot access Super Admin pages.

---

## UI Testing

Verify:

- Responsive Design
- Loading States
- Empty States
- Error States

---

## Security Testing

Verify:

- SQL Injection
- XSS
- CSRF
- Authentication

---

## Performance Testing

Dashboard Load

Target < 2 seconds

---

## Regression Testing

Existing functionality must never break.

Run regression tests after every module.

---

## Definition of Done

Feature complete only if:

✔ Tests passing

✔ No critical bugs

✔ No console errors

✔ No build errors
