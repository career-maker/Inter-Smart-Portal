# UI Rules

Project: Intersmart Employee Portal

---

## Design Philosophy

Create a premium enterprise SaaS experience.

Avoid traditional admin panel appearance.

Inspired by:

- Linear
- Slack
- Notion
- Stripe
- ClickUp
- Microsoft 365

---

## Global UI Standards

Use:

- Cards
- Widgets
- Charts
- Timelines
- Status Chips
- Progress Rings

Avoid:

- Plain tables everywhere
- Large unstructured forms
- Cluttered screens

---

## Layout

Desktop

- Left Sidebar
- Top Navigation
- Content Area

Mobile

- Collapsible Sidebar
- Bottom Navigation

---

## Components

All UI must use reusable components.

Examples:

- Card
- Modal
- Table
- Form
- Button
- Badge
- Chart
- Notification

---

## Dashboard Rules

Every dashboard shall contain:

- KPI Cards
- Charts
- Quick Actions
- Recent Activities
- Notifications

---

## Cards

Each card shall include:

- Icon
- Title
- Value
- Hover Effect
- Loading Skeleton

---

## Tables

Every table shall support:

- Search
- Sort
- Filter
- Pagination
- Export

---

## Forms

Every form shall support:

- Inline Validation
- Error Messages
- Success Messages
- Loading State

---

## Animations

Use subtle animations only.

Allowed:

- Fade
- Scale
- Slide
- Hover

Avoid excessive animations.

---

## Colors

Working → Green

WFH → Blue

Leave → Orange

Sick Leave → Red

Half Day → Purple

Pending → Yellow

Disabled → Gray

---

## Empty States

Every page must include:

- Illustration
- Message
- Action Button

---

## Loading States

Use Skeleton Loaders.

Never show blank screens.

---

## Responsiveness

Test all screens on:

- Desktop
- Tablet
- Mobile

---

## Accessibility

Support:

- Keyboard Navigation
- Focus States
- ARIA Labels
